# PRD: JSON 기반 할 일 목록 CRUD 콘솔 애플리케이션

## 개요

데이터를 JSON 파일로 관리하는 CRUD(Create, Read, Update, Delete) 콘솔 애플리케이션. 관리 대상 데이터는 할 일 목록(Todo)이다.

이 프로젝트에는 참고할 기존 PoC가 없으며, 이번 구현 자체가 최초 PoC가 된다. 최초 구현에서 확립되는 코드 구조(JSON 저장 계층 / CRUD 로직 / 콘솔 인터페이스의 분리 방식)는 이후 기능 추가 시에도 그대로 유지한다.

## 데이터 모델

할 일(Todo) 레코드는 다음 필드를 가진다.

| 필드 | 타입 | 설명 |
|---|---|---|
| id | 정수 | 레코드 식별자. 자동 증가(auto-increment)로 부여하며, 삭제된 ID는 재사용하지 않는다. |
| title | 문자열 | 할 일 제목 |
| description | 문자열 | 할 일 설명 |
| done | 불리언 | 완료 여부 |

## 기능 요구사항

### Create
- 사용자로부터 title, description을 입력받아 새 레코드를 생성한다.
- done은 생성 시 기본값 false로 설정한다.
- id는 자동 증가로 부여한다.
- 생성된 레코드를 JSON 파일에 저장한다.

### Read
- 전체 목록을 조회할 수 있다.
- 특정 id 또는 키 값(예: title)으로 검색할 수 있다.

### Update
- 기존 레코드를 (id 등으로) 선택하여 특정 필드(title, description, done)를 수정할 수 있다.

### Delete
- 특정 레코드를 id로 선택하여 삭제할 수 있다.
- 삭제 전 대상 존재 여부를 확인하는 등 안전하게 삭제한다(오삭제 방지).

## 환경

- 언어: Python 3.11
- 포맷터: Black
