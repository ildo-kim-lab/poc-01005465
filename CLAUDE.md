# CLAUDE.md

이 파일은 이 저장소에서 작업하는 Claude Code(claude.ai/code)에게 제공되는 안내 문서입니다.

## 프로젝트 상태

이 저장소는 아직 초기(코드 이전) 상태입니다 — 현재 소스 파일은 없고 IDE/프로젝트 메타데이터(`.idea/`)만 존재합니다. 빌드 시스템, 테스트, 린트 설정도 아직 없습니다.

## 프로젝트 목표

데이터를 JSON 파일에 저장하는 CRUD(Create, Read, Update, Delete) 콘솔 애플리케이션.

참고할 기존 PoC는 없으며, 이 저장소의 최초 구현 자체가 PoC가 됩니다. 최초 구현에서 확립되는 코드 구조와 로직(모듈 구성, JSON 저장 계층과 CRUD 로직 및 콘솔 인터페이스의 분리 방식)을 기준으로 삼아야 합니다. 이후 기능을 추가할 때 구조를 새로 바꾸기보다는, 확립된 구조 안에서 확장하고 병렬적이거나 경쟁하는 패턴을 도입하지 않아야 합니다.

## 요구사항 (docs/PRD.md 참고)

관리 대상 데이터는 할 일(Todo) 목록이며, 필드는 id(자동 증가 정수), title, description, done(불리언)입니다.

- Create: title/description을 입력받아 새 레코드 생성 (id 자동 증가, done 기본값 false)
- Read: 전체 목록 조회, id 또는 키 값으로 검색
- Update: id로 레코드를 선택해 특정 필드 수정
- Delete: id로 레코드를 선택해 삭제 (삭제 전 존재 여부 확인 등 안전하게 처리)

## 알려진 환경

- 언어: Python (`.idea/misc.xml` / `.idea/PoC.iml`에 Python 3.11로 설정됨)
- 포맷터: Black (`.idea/misc.xml`에 설정됨)

## 향후 갱신 시 참고

소스 파일, 패키지 구조, 테스트/린트 도구가 갖춰지면 이 파일에 다음 내용을 추가로 반영합니다.
- 실제 실행/빌드/린트/테스트 명령어 (단일 테스트 실행 방법 포함)
- CRUD 로직, JSON 저장 계층, 콘솔 인터페이스가 실제로 어떻게 연결되는지에 대한 모듈 구조 — 이 구조가 이후에도 유지해야 할 기준이 됩니다.
